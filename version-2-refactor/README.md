# Geometric Figures – Object-Oriented Version

## Description

This project represents the refactored and improved version of the initial geometric figures assignment.

Unlike the first version, this implementation applies Object-Oriented Programming principles to improve structure, scalability, and code readability.

The application allows the user to calculate the area and perimeter of different geometric figures through a console-based interactive menu.

## Improvements Over Initial Version

- Introduction of an abstract class `Figura`
- Use of inheritance for each geometric figure
- Method overriding (`@Override`)
- Polymorphism
- Separation of responsibilities
- Cleaner and more maintainable structure

## Project Structure

- `Figura` → Abstract base class
- `Circulo`, `Cuadrado`, `Rectangulo`, `Triangulo` → Concrete implementations
- `MenuOpciones` → Handles menu logic
- `RecogeDatos` → Manages user input
- `Resultado` → Displays output
- `MensajeYCondicion` → Input validation logic
- `Principal` → Application entry point

## Learning Objectives

This version demonstrates:

- Object-Oriented design
- Abstraction
- Encapsulation
- Code modularization
- Responsibility separation

The goal of this refactor was to transition from structured programming to a scalable OOP design.

# Figuras Geométricas – Versión Orientada a Objetos

## Descripción

Este proyecto representa la versión refactorizada y mejorada de la actividad inicial de figuras geométricas.

A diferencia de la primera versión, esta implementación aplica principios de Programación Orientada a Objetos para mejorar la estructura, escalabilidad y legibilidad del código.

La aplicación permite al usuario calcular el área y el perímetro de diferentes figuras geométricas mediante un menú interactivo por consola.

## Mejoras respecto a la versión inicial

- Introducción de una clase abstracta `Figura`
- Uso de herencia para cada figura geométrica
- Sobreescritura de métodos (`@Override`)
- Polimorfismo
- Separación de responsabilidades
- Estructura más limpia y mantenible

## Estructura del Proyecto

- `Figura` → Clase base abstracta
- `Circulo`, `Cuadrado`, `Rectangulo`, `Triangulo` → Implementaciones concretas
- `MenuOpciones` → Gestiona la lógica del menú
- `RecogeDatos` → Maneja la entrada de datos del usuario
- `Resultado` → Muestra los resultados
- `MensajeYCondicion` → Lógica de validación de datos
- `Principal` → Punto de entrada de la aplicación

## Objetivos de Aprendizaje

Esta versión demuestra:

- Diseño Orientado a Objetos
- Abstracción
- Encapsulación
- Modularización del código
- Separación de responsabilidades

El objetivo de esta refactorización fue realizar la transición desde programación estructurada hacia un diseño orientado a objetos más escalable.
